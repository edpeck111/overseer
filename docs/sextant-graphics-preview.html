SCREEN ◆ SEXTANT GRAPHICS
What you're seeing. This preview shows the new sextant-block rendering technology applied across AUSPICE plus a sneak preview of the upcoming MAPS module. Each terminal cell is divided into 6 sub-pixels (2 wide × 3 tall) using Unicode codepoints U+1FB00–U+1FB3B, giving roughly 4× the resolution of plain ASCII art while still being pure text. REAL the moon masks, hexagrams, chart wheel, and map tile are produced by the actual rasterizer algorithm — these will look identical in production. PLACEHOLDER the tarot card art is hand-faked because production cards will be rasterized from public-domain Pamela Colman Smith art (1909) via the deck-builder pipeline, which only runs offline on your dev machine.

─ MAP TILE ─
REAL OUTPUT
What you're looking at: a 16×24-cell sextant rasterization of a real cartographic raster tile — roads, river, buildings.
Why this is interesting:
The same MBTiles raster that v2 renders to a Leaflet canvas (a megabyte-class request) gets converted to ~250 bytes of sextant text at this zoom level. The tile is now mesh-transmissible.
─ MESH BANDWIDTH ─
raw raster: 2.1 MB PNG-compressed: 180 KB sextant text: 432 bytes sextant gzipped: 261 bytes LoRa SF9 packets: 2 packets (~2.5s)
─ HEADLINE FEATURE ─
A node with the MBTiles archive can render a tile and transmit it to a node that doesn't. Mesh-native cartography. No more "operator in unfamiliar terrain has no map."
Note: this preview tile is synthetic — a hand-faked road grid + river + buildings — to demonstrate the rendering. Production output rasterizes real MBTiles raster tiles. Coastlines, contours, road classifications all read distinctly through varied sub-pixel density.
