O.V.E.R.S.E.E.R.

Offline Vault of Essential Records for Survival, Emergency & Endurance Response

Version 2.0.0 — Setup & Operations Guide

1. WHAT IS OVERSEER?

A fully offline, battery-powered platform running on an Orange Pi 5 Max. It provides:

  • KNOWLEDGE — LLM chat (Qwen 2.5 7B) augmented with offline Kiwix knowledge archives (Wikipedia, medical, repair guides, etc.)
  • LIBRARY — Browse and read full articles from all loaded ZIM archives
  • COMMS — Encrypted store-and-forward messaging between registered operators
  • RECREATION — Text-based games (coming soon)
  • SYSTEM — Status monitoring and admin panel

Users connect via phone browser over WiFi. Future: Pi Zero relays over LoRa/Meshtastic.

2. FRESH DEPLOYMENT (Orange Pi 5 Max)

Prerequisites

  • Orange Pi 5 Max with Armbian installed on NVMe
  • Internet connection (for initial setup only)
  • ~50GB free disk space (more for Wikipedia)

Setup

git clone https://github.com/edpeck111/overseer.git
cd overseer
chmod +x setup.sh
./setup.sh

The script will automatically:

  • Install Python, pip, system packages
  • Install Flask, requests, psutil, cryptography
  • Install Ollama and pull qwen2.5:7b-instruct-q4_K_M
  • Download kiwix-serve (auto-detects aarch64)
  • Download all 19 ZIM archives (~28GB) with resume support
  • Create Linux startup scripts
  • Generate a systemd service file

Start manually

./start_overseer.sh

Install as boot service

sudo cp /tmp/overseer.service /etc/systemd/system/
sudo systemctl enable overseer
sudo systemctl start overseer

Server runs on http://<device-ip>:6100

3. ADMIN SETUP

First login

  1. Open browser, go to http://<device-ip>:6100
  2. Click to initialize the terminal
  3. Navigate to SYSTEM tab, then ADMIN sub-tab
  4. Enter default PIN: 1234
  5. Change the PIN immediately via the "Change Admin PIN" section

The SYSADMIN account

A SYSADMIN user is auto-created on first boot using the keypair in keys/. The private key (keys/admin_private.pem) stays on the OPi5.

4. REGISTERING OPERATORS

For each new user:

  1. Go to SYSTEM > ADMIN (enter PIN)
  2. Type a callsign (e.g. BRAVO-1)
  3. Click GENERATE KEYPAIR
  4. The private key auto-downloads as bravo-1_private_key.pem — save this securely
  5. The public key fills in automatically
  6. Click REGISTER

The operator can now log in to COMMS by selecting their callsign.

5. COMMS — MESSAGING

Logging in

  1. Navigate to the COMMS tab
  2. Select your callsign from the operator list

Sending a message

  1. Click the COMPOSE tab
  2. Select a recipient from the dropdown
  3. Enter a subject (optional) and message body
  4. Click TRANSMIT

Reading messages

  1. Click INBOX — unread messages show a green left border and a badge count
  2. Click a message to read it (auto-marks as read)
  3. Click < BACK to return to the inbox

Checking sent messages

Click the SENT tab to see all messages you've transmitted.

Logging out

Click [DISCONNECT] in the top bar to return to the operator selection screen.

6. KNOWLEDGE — LLM CHAT

Chat sub-tab

  • Type a question and click TRANSMIT (or press Enter)
  • The LLM searches Kiwix archives for relevant context (RAG) and responds
  • Toggle [KNOWLEDGE BASE AUGMENTATION] checkbox to disable RAG
  • [CLR] clears chat history, [EXPORT] saves to .txt file
  • Chat history persists in browser localStorage

Library sub-tab

  • SEARCH — full-text search across all loaded ZIM archives
  • BROWSE — lists all loaded volumes, click to open a volume's main page
  • Click any article to read it — headings are styled for readability
  • LINKED ARTICLES panel at the bottom shows navigable links from the current article (drag the handle to resize)
  • Click < BACK to return to search results

7. PI ZERO CLIENT SETUP (Future)

The end-goal architecture:

Phone (WiFi) --> Pi Zero (client relay) --> LoRa/Meshtastic --> OPi5 (server)

Provisioning a Pi Zero

  1. Register the operator on the OPi5 (see Section 4)
  2. Flash the Pi Zero with the client image
  3. Copy the operator's private key to: /home/overseer/keys/identity.pem
  4. The Pi Zero reads identity.pem on boot to authenticate with the OPi5
  5. It serves a local WiFi AP for the phone to connect to
  6. Messages are queued and synced when in LoRa range of the OPi5

8. KEY MANAGEMENT

keys/
  admin_private.pem    — OPi5 system admin key (stays on OPi5)
  admin_public.pem     — Auto-registered as SYSADMIN user
  <callsign>_private.pem — Per-user keys (burn onto Pi Zero)
  • Ed25519 keypairs — small, fast, quantum-resistant-adjacent
  • Private keys NEVER leave the device they belong to (except during provisioning)
  • The OPi5 only stores public keys in the database
  • Future: messages encrypted end-to-end with recipient's public key

IMPORTANT: Back up keys/admin_private.pem somewhere safe. If lost, you'll need to regenerate and re-register all operators.

11. TROUBLESHOOTING

Kiwix server unreachableKiwix isn't running. Start it with start_kiwix.sh or check for corrupt ZIM files.
Ollama connection refusedRun ollama serve in a separate terminal, then ollama pull qwen2.5:7b-instruct-q4_K_M
ZIM file won't loadLikely a truncated download. Delete it and re-run setup.sh (it resumes/re-downloads).
Garbled characters in articlesUTF-8 encoding issue — already fixed in v2.0.0. Ensure you're on latest code.
Forgot admin PINDelete overseer.db and restart. DB will be recreated with default PIN 1234.
